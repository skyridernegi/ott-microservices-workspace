Phase 1 — Build JARs and create Dockerfiles

WHY JAR first: Docker cannot build your Spring Boot app from source code directly. You first build a JAR (Java executable) using Maven, then Docker packages that JAR into an image. Think of it as: Maven builds the car, Docker puts it in a shipping container.
1Build JAR — do this for ALL 3 services in STS

In STS for each service:

Right-click project name (e.g. catalog-service)
→ Run As
→ Maven build...
→ In Goals field type: clean package -DskipTests
→ Click Run
Wait for BUILD SUCCESS in STS console. Then verify JAR was created:
catalog-service → target → catalog-service-1.0.0.jar ✅
subscription-service → target → subscription-service-1.0.0.jar ✅
api-gateway → target → api-gateway-1.0.0.jar ✅
If JAR name is different (e.g. catalog-service-0.0.1-SNAPSHOT.jar) — note the exact name. You'll use it in Dockerfile below.
2Create Dockerfile — catalog-service
Dockerfile NEW FILE catalog-service\ (same folder as pom.xml — NOT inside src)

HOW TO CREATE IN WINDOWS: Open Notepad → paste code below → File → Save As → navigate to catalog-service folder → filename: Dockerfile (no extension) → Save as type: All Files → Save

WHY each line:

# FROM — base image. eclipse-temurin:21-jre-alpine is Java 21
# slim Alpine Linux. Small size (~200MB vs ~500MB for full JDK)
FROM eclipse-temurin:21-jre-alpine

# WORKDIR — working directory inside container
# All subsequent commands run from this folder
WORKDIR /app

# COPY — copies JAR from your Windows machine into container
# Left side: path on Windows (target/catalog-service-1.0.0.jar)
# Right side: path inside container (app.jar)
# Update the JAR name if yours is different
COPY target/catalog-service-1.0.0.jar app.jar

# EXPOSE — documents which port this container listens on
# Does NOT actually open the port — that happens in docker-compose
EXPOSE 8081

# ENTRYPOINT — command that runs when container starts
# -Djava.security.egd speeds up Spring Boot startup
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","app.jar"]
3Create Dockerfile — subscription-service
Dockerfile NEW FILE subscription-service\ (same folder as pom.xml) 
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY target/subscription-service-1.0.0.jar app.jar
EXPOSE 8082
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","app.jar"]
4Create Dockerfile — api-gateway
Dockerfile NEW FILE api-gateway\ (same folder as pom.xml) 
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY target/api-gateway-1.0.0.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","app.jar"]
5Create docker-compose.yml — runs everything together
docker-compose.yml NEW FILE ott-platform\ (parent folder — same level as all 3 service folders)

WHY docker-compose: Starts MySQL, Redis, and all 3 Spring Boot services in correct order with networking between them — one command instead of 5 separate starts.

version: '3.8'

services:

  # ── MySQL ─────────────────────────────────────
  mysql:
    image: mysql:8.0
    container_name: ott-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: catalog_db
    ports:
      - "3307:3306"
      # 3307 on Windows to avoid conflict with your local MySQL on 3306
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - ott-network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-uroot", "-proot"]
      interval: 10s
      timeout: 5s
      retries: 10

  # ── Redis ──────────────────────────────────────
  redis:
    image: redis:7-alpine
    container_name: ott-redis
    ports:
      - "6380:6379"
      # 6380 on Windows to avoid conflict with your local Redis on 6379
    networks:
      - ott-network

  # ── Catalog Service ────────────────────────────
  catalog-service:
    build:
      context: ./catalog-service
    container_name: ott-catalog
    ports:
      - "8081:8081"
    environment:
      # Override localhost with Docker container names
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/catalog_db?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root
      SPRING_JPA_HIBERNATE_DDL_AUTO: update
    depends_on:
      mysql:
        condition: service_healthy
    networks:
      - ott-network

  # ── Subscription Service ───────────────────────
  subscription-service:
    build:
      context: ./subscription-service
    container_name: ott-subscription
    ports:
      - "8082:8082"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/subscription_db?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root
      SPRING_JPA_HIBERNATE_DDL_AUTO: update
      # Use container name instead of localhost for Feign calls
      CATALOG_SERVICE_URL: http://catalog-service:8081
    depends_on:
      mysql:
        condition: service_healthy
      catalog-service:
        condition: service_started
    networks:
      - ott-network

  # ── API Gateway ────────────────────────────────
  api-gateway:
    build:
      context: ./api-gateway
    container_name: ott-gateway
    ports:
      - "8080:8080"
    environment:
      # Use Redis container name instead of localhost
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_PORT: 6379
    depends_on:
      - redis
      - catalog-service
      - subscription-service
    networks:
      - ott-network

Phase 2 — Test locally with Docker before pushing to AWS

WHY test locally first: Always verify Docker works on your machine before pushing to AWS. Fixing errors locally takes seconds. Fixing on AWS takes minutes and uses your free tier quota.
1Open Command Prompt — navigate to ott-platform folder
REM Navigate to your ott-platform folder
REM Replace with your actual path
cd C:\Users\skyri\Desktop\ott-platform
2Build and start all containers
REM Build images and start all services
REM --build forces rebuild even if image already exists
docker compose up --build

REM This will take 3-5 minutes first time
REM You'll see logs from all services streaming
Look for these lines in the output: ott-catalog | Started CatalogServiceApplication on port 8081 ✅ ott-subscription | Started SubscriptionServiceApplication on port 8082 ✅ ott-gateway | Started ApiGatewayApplication on port 8080 ✅
If you see "port already in use" — stop your STS services first. Docker and STS cannot both use the same port.
3Verify all containers are running

Open a NEW Command Prompt window (keep the first one running) and type:

docker compose ps
You should see all 5 containers with status "running": ott-mysql running ott-redis running ott-catalog running ott-subscription running ott-gateway running

5Useful Docker commands to know
REM View logs for one service
docker compose logs catalog-service

REM Follow logs in real time
docker compose logs -f api-gateway

REM Stop all containers (keeps data)
docker compose stop

REM Stop and remove containers (keeps data volumes)
docker compose down

REM Stop, remove containers AND delete all data
docker compose down -v

docker compose restart catalog-service

Phase 3 — Set up AWS CLI on Windows 10

1Download and install AWS CLI

Go to: https://aws.amazon.com/cli → click Download → download AWSCLIV2.msi for Windows → run installer → all defaults → finish

Verify in new Command Prompt:

aws --version
Should show: aws-cli/2.x.x Python/3.x.x Windows/10
2Create IAM user in AWS Console — get Access Keys

WHY IAM user: Never use your root AWS account for CLI. IAM (Identity and Access Management) lets you create a user with specific permissions. Best practice for security.

  1. Open https://console.aws.amazon.com → login with your account
  2. Search for IAM in top search bar → click IAM
  3. Left menu → Users → click Create user
  4. Username: ott-deployer → Next
  5. Permissions: click Attach policies directly → search and select:
    • AmazonEC2FullAccess
    • AmazonECR_FullAccess (or AmazonEC2ContainerRegistryFullAccess)
  6. Click Create user
  7. Click on the new user → Security credentials tab → Create access key
  8. Use case: CLI → Next → Create
  9. DOWNLOAD the CSV file or copy both keys NOW — you cannot see Secret key again!
3Configure AWS CLI with your credentials

Open Command Prompt and run:

aws configure

Enter values when prompted:

AWS Access Key ID:     [paste your Access Key ID]
AWS Secret Access Key: [paste your Secret Access Key]
Default region name:   ap-south-1
Default output format: json

Verify it works:

aws sts get-caller-identity
Should show your AWS Account ID and username. If you see this — AWS CLI is configured correctly.
If you see "credentials not found" — re-run aws configure and paste keys carefully.

Phase 4 — Push Docker images to AWS ECR

WHY ECR: ECR (Elastic Container Registry) is AWS's private Docker image storage. Your EC2 server will pull images from ECR. Like a private Docker Hub but inside AWS — faster, secure, and integrated with EC2.
1Get your AWS Account ID
aws sts get-caller-identity --query Account --output text

This shows your 12-digit Account ID like: 123456789012

Copy it — you need it in every command below. Replace YOUR_ACCOUNT_ID with this number.

2Create ECR repositories

Run these 3 commands — creates a repository for each service:

aws ecr create-repository --repository-name ott/catalog-service --region ap-south-1

aws ecr create-repository --repository-name ott/subscription-service --region ap-south-1

aws ecr create-repository --repository-name ott/api-gateway --region ap-south-1
Each command shows a JSON response with "repositoryUri". That's the address of your image storage.
3Login to ECR from Docker

Replace YOUR_ACCOUNT_ID with your actual account ID from Step 1:

aws ecr get-login-password --region ap-south-1 | docker login --username AWS --password-stdin YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com
Must see: Login Succeeded ✅ — if not, check your IAM permissions.
4Build, tag and push all images to ECR

Run from your ott-platform folder. Replace YOUR_ACCOUNT_ID in each command:

REM ── catalog-service ──────────────────────────────
docker build -t ott/catalog-service ./catalog-service

docker tag ott/catalog-service:latest YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com/ott/catalog-service:latest

docker push YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com/ott/catalog-service:latest

REM ── subscription-service ─────────────────────────
docker build -t ott/subscription-service ./subscription-service

docker tag ott/subscription-service:latest YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com/ott/subscription-service:latest

docker push YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com/ott/subscription-service:latest

REM ── api-gateway ──────────────────────────────────
docker build -t ott/api-gateway ./api-gateway

docker tag ott/api-gateway:latest YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com/ott/api-gateway:latest

docker push YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com/ott/api-gateway:latest
Each push shows progress bars. When done — verify in AWS Console → ECR → Repositories → you should see all 3 repos with images.

Phase 5 — Launch EC2 instance on AWS

WHY EC2: EC2 (Elastic Compute Cloud) is a virtual server on AWS. We'll run Docker on this server and pull our images from ECR to run the OTT platform publicly accessible on the internet.
1Launch EC2 in AWS Console — step by step

AWS Console → search EC2 → click EC2 → click Launch Instance:

FieldValueWhy
Nameott-platform-serverEasy to identify
AMIAmazon Linux 2023 (free tier eligible)AWS native, has yum package manager
Instance typet2.microFree tier — 1 CPU, 1GB RAM
Key pairCreate new → name: ott-key → RSA → .pem → DownloadNeeded for SSH access
Storage20 GB gp2Free tier allows 30GB
Save ott-key.pem file somewhere safe — like C:\Users\skyri\aws-keys\ — you CANNOT download it again after this screen.
2Configure Security Group — open required ports

On the same Launch Instance page → Network settings → click Edit:

Click Add security group rule for each row below:

TypePortSourcePurpose
SSH22My IPConnect to server
Custom TCP8080Anywhere (0.0.0.0/0)API Gateway
Custom TCP8081Anywhere (0.0.0.0/0)Catalog Service
Custom TCP8082Anywhere (0.0.0.0/0)Subscription Service

Click Launch Instance → wait 1-2 minutes → Instance State: Running ✅

Note your Public IPv4 address — you'll need it to SSH and test.

3Attach IAM role to EC2 — allows pulling from ECR

WHY: EC2 needs permission to pull images from ECR. Instead of putting credentials on the server, we attach an IAM Role — AWS automatically provides credentials.

  1. AWS Console → IAM → Roles → Create role
  2. Trusted entity: AWS service → Use case: EC2 → Next
  3. Search and select: AmazonEC2ContainerRegistryReadOnly → Next
  4. Role name: ott-ec2-ecr-role → Create role
  5. Go to EC2 → select your instance → Actions → Security → Modify IAM role
  6. Select ott-ec2-ecr-role → Save
Now your EC2 can pull from ECR without any access keys on the server.

Phase 6 — Connect to EC2 and deploy

1Connect to EC2 via SSH from Windows Command Prompt

Open Command Prompt → navigate to folder where ott-key.pem is saved → run:

REM First fix key file permissions (Windows 10)
icacls ott-key.pem /inheritance:r
icacls ott-key.pem /grant:r "%USERNAME%:R"

REM Connect to EC2 — replace YOUR_EC2_PUBLIC_IP
ssh -i ott-key.pem ec2-user@YOUR_EC2_PUBLIC_IP

Type yes when asked about fingerprint. You'll see:

   ,     #_
   ~\_  ####_        Amazon Linux 2023
  ~~  \_#####\
[ec2-user@ip-10-0-1-x ~]$  ← you are now inside EC2!
2Install Docker on EC2 — run these commands inside EC2
You are now typing inside the EC2 server — NOT your Windows machine. These are Linux commands.
# Update packages
sudo dnf update -y

# Install Docker
sudo dnf install docker -y

# Start Docker service
sudo systemctl start docker
sudo systemctl enable docker

# Add ec2-user to docker group (no need for sudo)
sudo usermod -a -G docker ec2-user

# Install Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Logout and reconnect so group change takes effect
exit

Then reconnect:

ssh -i ott-key.pem ec2-user@YOUR_EC2_PUBLIC_IP
3Login to ECR from EC2 and create docker-compose

Inside EC2 terminal — replace YOUR_ACCOUNT_ID:

# Login to ECR — EC2 uses IAM role, no credentials needed
aws ecr get-login-password --region ap-south-1 | docker login --username AWS --password-stdin YOUR_ACCOUNT_ID.dkr.ecr.ap-south-1.amazonaws.com

# Create folder for our app
mkdir ott-platform
cd ott-platform

nano docker-compose.yml

7IMPORTANT — Stop EC2 when not using to save free tier

Free tier gives you 750 hours/month of t2.micro. That's enough for one instance running all month. But stop it when not needed:

AWS Console → EC2 → Instances → select ott-platform-server
→ Instance state → Stop instance
STOP not TERMINATE. Stop = like shutting down (data preserved). Terminate = permanent deletion.

When you want to use it again → Start instance → wait 1 min → new Public IP will be assigned → update Postman URLs.

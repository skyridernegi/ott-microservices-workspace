Every Java file and what it does

☕ ApiGatewayApplication.java com.ott.gateway

The entry point. Spring Boot starts here. The single important annotation is @SpringBootApplication which triggers a component scan — Spring goes through every class in com.ott.gateway and registers anything annotated with @Component, @Service, @RestController etc.

This is how JwtAuthFilter and LoggingFilter get registered automatically — they have @Component, so Spring finds them during startup.

@SpringBootApplication
public class ApiGatewayApplication {
    public static void main(String[] args) {
        SpringApplication.run(ApiGatewayApplication.class, args);
    }
}
☕ JwtAuthFilter.java com.ott.gateway.filter

The security guard. Runs on every single request automatically because it implements GlobalFilter. It either lets the request pass or returns 401. It does NOT need to be mentioned in application.yml — Spring Cloud Gateway picks it up automatically via @Component.

Contains the PUBLIC_PATHS list — the only place where you say which URLs skip JWT checking.

☕ LoggingFilter.java com.ott.gateway.filter

The logger. Also a GlobalFilter, also runs on every request automatically. Logs the method, path, and response status to the STS console. Runs at @Order(2) — after JwtAuthFilter.

☕ JwtUtil.java com.ott.gateway.util

The JWT toolkit. Called by JwtAuthFilter to validate tokens, and by AuthController to generate tokens. Contains three key methods: generateToken(), isTokenValid(), extractUsername().

☕ AuthController.java com.ott.gateway.controller

Exposes POST /auth/token and POST /auth/validate. Uses JwtUtil to generate a token and return it. This is the only REST controller inside the Gateway itself — all other URLs are proxied to downstream services.

☕ RateLimiterConfig.java com.ott.gateway.config

Defines the userKeyResolver bean — tells the rate limiter to count requests per username (read from X-Username header). Referenced in application.yml as key-resolver: "#{@userKeyResolver}".

⚙️ application.yml src/main/resources

Defines routing rules only — which URL path goes to which service port. Also configures the RequestRateLimiter per route. Does NOT control JWT protection — that is handled entirely by JwtAuthFilter in Java code.

Complete request flow — which Java file runs in which order

Every request to port 8080 goes through this exact sequence of Java files regardless of which URL is called.
1 Spring's DispatcherHandler Spring framework internal class

The reactive HTTP server receives the request. Hands it to FilteringWebHandler which assembles the filter chain. You don't write this — it's Spring internals.

↓
2 LoggingFilter.java — YOUR FILE com.ott.gateway.filter · @Order(2) · GlobalFilter

Runs first in terms of wrapping (it wraps the entire chain). Logs the incoming request method and path. Then calls chain.filter(exchange) to pass control to the next filter. When the response comes back, logs the status code and time taken.

// LoggingFilter wraps the whole chain
return chain.filter(exchange).then(Mono.fromRunnable(() -> {
    // This runs AFTER the response comes back
    log.info("◄── RESPONSE | status: {} | time: {}ms", status, duration);
}));
↓
3 JwtAuthFilter.java — YOUR FILE com.ott.gateway.filter · @Order(1) · GlobalFilter

The most important filter. Runs on every request. Makes the allow/block decision. Calls JwtUtil to validate the token.

Decision logic inside this file:

// Step 1: Is this a public path?
if (isPublicPath(path)) {
    return chain.filter(exchange); // pass through — no token needed
}

// Step 2: Is Authorization header present?
String authHeader = request.getHeaders().getFirst("Authorization");
if (authHeader == null || !authHeader.startsWith("Bearer ")) {
    exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
    return exchange.getResponse().setComplete(); // STOP — 401
}

// Step 3: Is the token valid?
String token = authHeader.substring(7);
if (!jwtUtil.isTokenValid(token)) {
    exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
    return exchange.getResponse().setComplete(); // STOP — 401
}

// Step 4: Token valid — add username to headers and continue
return chain.filter(modifiedExchange);

Calls JwtUtil.isTokenValid() in Step 3 — that's the only connection between these two files.

↓ (only reaches here if token was valid or path is public)
4 RequestRateLimiter — Spring's built-in filter application.yml configures this · GatewayFilter · only on /catalog and /subscriptions routes

Checks Redis to see how many requests this user has made. Uses userKeyResolver bean from RateLimiterConfig.java to identify the user by their X-Username header. If over limit → 429. If within limit → continue.

↓ (only reaches here if within rate limit)
5 Spring's NettyRoutingFilter Spring framework internal — last filter in chain

The actual HTTP proxy. Makes the real call to catalog-service:8081 or subscription-service:8082 based on the routing rules in application.yml. Returns the response back through the filter chain.

Only YOUR files are JwtAuthFilter, LoggingFilter, RateLimiterConfig, JwtUtil, AuthController. Everything else is Spring framework internals that run automatically.

JwtAuthFilter.java — complete line by line

Full annotated code com.ott.gateway.filter.JwtAuthFilter 
@Component   // Spring registers this as a bean automatically
@Order(1)    // Run before LoggingFilter (@Order 2)
public class JwtAuthFilter implements GlobalFilter, Ordered {

    // ─────────────────────────────────────────────────
    // PUBLIC_PATHS — THE ONLY PLACE IN THE ENTIRE
    // CODEBASE WHERE YOU DEFINE WHICH URLS ARE PUBLIC
    // ─────────────────────────────────────────────────
    // Any URL that STARTS WITH these strings will skip
    // JWT validation. All other URLs require a valid token.
    // To make a new URL public: just add it to this list.
    private static final List<String> PUBLIC_PATHS = List.of(
        "/auth/token",    // POST — generate token
        "/auth/refresh"   // POST — refresh token
    );

    private final JwtUtil jwtUtil; // injected — validates tokens

    @Override
    public Mono<Void> filter(ServerWebExchange exchange,
                              GatewayFilterChain chain) {

        String path = exchange.getRequest().getPath().toString();

        // GATE 1: Is this path in PUBLIC_PATHS?
        // YES → skip all JWT logic, forward immediately
        if (isPublicPath(path)) {
            return chain.filter(exchange);
        }

        // GATE 2: Does Authorization header exist?
        // Format must be: "Bearer eyJhbG..."
        String authHeader = exchange.getRequest()
            .getHeaders().getFirst("Authorization");

        if (authHeader == null ||
            !authHeader.startsWith("Bearer ")) {
            exchange.getResponse()
                .setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
            // REQUEST STOPS HERE — 401 returned
        }

        // GATE 3: Is the token cryptographically valid?
        // Calls JwtUtil — checks signature + expiry
        String token = authHeader.substring(7); // removes "Bearer "
        if (!jwtUtil.isTokenValid(token)) {
            exchange.getResponse()
                .setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
            // REQUEST STOPS HERE — 401 returned
        }

        // ALL GATES PASSED — extract user info
        // Add X-Username and X-Role headers so downstream
        // microservices know who is calling them
        String username = jwtUtil.extractUsername(token);
        String role = jwtUtil.extractRole(token);

        ServerHttpRequest modifiedRequest = exchange.getRequest()
            .mutate()
            .header("X-Username", username)
            .header("X-Role", role)
            .build();

        return chain.filter(
            exchange.mutate().request(modifiedRequest).build()
        );
        // REQUEST FORWARDED TO NEXT FILTER / MICROSERVICE
    }

    private boolean isPublicPath(String path) {
        return PUBLIC_PATHS.stream()
            .anyMatch(path::startsWith);
    }
}

How JwtUtil.java connects to JwtAuthFilter

JwtUtil.java — methods called by JwtAuthFilter com.ott.gateway.util.JwtUtil 
// Called from JwtAuthFilter Gate 3
public boolean isTokenValid(String token) {
    try {
        Jwts.parserBuilder()
            .setSigningKey(getSigningKey()) // uses jwt.secret from yml
            .build()
            .parseClaimsJws(token);         // throws if expired or tampered
        return true;
    } catch (Exception e) {
        return false; // any error = invalid token
    }
}

// Called from JwtAuthFilter after token is valid
public String extractUsername(String token) {
    return extractClaims(token).getSubject();
}

// Called from AuthController to create a new token
public String generateToken(String username, String role) {
    return Jwts.builder()
        .setSubject(username)
        .claim("role", role)
        .setExpiration(new Date(System.currentTimeMillis() + expiration))
        .signWith(getSigningKey(), SignatureAlgorithm.HS256)
        .compact();
}

Your exact question — why is /auth/** accessible without a token?

You asked: "JwtAuthFilter runs on /auth/** — so why is /auth/token still accessible without a token?"

This is a really important question. Here is the complete honest answer.
The answer: JwtAuthFilter runs on /auth/token BUT it checks PUBLIC_PATHS first and exits early

Yes, JwtAuthFilter DOES run on every request including /auth/token. There is no way to stop a GlobalFilter from running on a route — it always runs. But running does not mean blocking. The filter looks at the path, sees it is in PUBLIC_PATHS, and immediately calls chain.filter(exchange) to pass through without checking the token.

Step by step for POST /auth/token:

1
Request arrives: POST /auth/token — no Authorization header
↓
2
JwtAuthFilter.filter() is called — it ALWAYS runs
↓
3
isPublicPath("/auth/token") is called
↓
4
PUBLIC_PATHS contains "/auth/token" → method returns true
↓
5
return chain.filter(exchange) — skips all JWT checks, passes through
↓
6
Request reaches AuthController.generateToken() → token generated → 200 OK
So the filter DOES run — it just exits at Gate 1 without doing anything. Think of it like a security guard who checks your badge but has a list of VIP guests who can walk in without a badge. The guard still stands at the door for everyone — he just waves through the VIPs.
public Mono<Void> filter(...) {
    String path = exchange.getRequest().getPath().toString();
    // path = "/auth/token"

    if (isPublicPath(path)) {        // TRUE for /auth/token
        return chain.filter(exchange); // exits HERE — no token check at all
    }
    // ↑ The code below this line never runs for /auth/token

    String authHeader = ...         // NEVER reached for /auth/token
    jwtUtil.isTokenValid(token);    // NEVER reached for /auth/token
}

What should have been explained to you from the start

The complete mental model
URLJwtAuthFilter runs?Token checked?Why
POST /auth/token YES runs NO check In PUBLIC_PATHS → exits at Gate 1
POST /auth/validate YES runs NO check In PUBLIC_PATHS → exits at Gate 1
GET /catalog/plans YES runs YES checked NOT in PUBLIC_PATHS → reaches Gate 2 + Gate 3
POST /subscriptions YES runs YES checked NOT in PUBLIC_PATHS → reaches Gate 2 + Gate 3
GET /users/1 YES runs YES checked NOT in PUBLIC_PATHS → reaches Gate 2 + Gate 3
Rule: JwtAuthFilter runs on 100% of requests. Whether it checks the token depends entirely on whether the path is in PUBLIC_PATHS.

Interview answer — "how does your gateway know which URLs need JWT?"

Say exactly this

"JWT validation is handled by a GlobalFilter called JwtAuthFilter. Because it implements GlobalFilter with @Component, Spring Cloud Gateway automatically runs it on every single request without any yml configuration. Inside the filter, I maintain a PUBLIC_PATHS list — any URL that starts with a path in that list exits the filter immediately without token checking. All other URLs go through three gates: check for Authorization header, check it starts with Bearer, and validate the token signature and expiry via JwtUtil. If any gate fails, the filter returns 401 and the request never reaches the downstream microservice."

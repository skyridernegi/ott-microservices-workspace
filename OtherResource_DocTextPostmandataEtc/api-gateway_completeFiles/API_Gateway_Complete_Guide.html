1 What is API Gateway? (Interview Answer)

API Gateway is the single entry point for all client requests. Instead of the ESPN website calling catalog-service:8081 and subscription-service:8082 directly, everything goes through the Gateway on port 8080.

What Gateway does

  • Routing — /catalog → 8081, /subscriptions → 8082
  • JWT Auth — validates token before forwarding
  • Rate Limiting — max 10 req/sec per user
  • Logging — logs every request centrally

What JWT is (interview answer)

JSON Web Token — 3 parts separated by dots:

Header.Payload.Signature
eyJhbG..  eyJ1c2..  SflKx..
↑ algo    ↑ data    ↑ proof not tampered

Stateless — user data is IN the token. No server session needed.

9 Top Interview Questions — Answers Ready

Q: What is API Gateway and why do we need it?

API Gateway is a single entry point for all client requests. Without it, the client must know addresses of all microservices. Gateway handles cross-cutting concerns like authentication, rate limiting, and logging in one place instead of duplicating in every service.

Q: How does JWT authentication work in your gateway?

Every request hits JwtAuthFilter first. It checks the Authorization header for "Bearer token". It validates the token using HMAC-SHA256 signature. If valid, it extracts the username and forwards it as X-Username header to the downstream service. If invalid, it returns 401 immediately — request never reaches the microservice.

Q: What is rate limiting and how did you implement it?

Rate limiting controls how many requests a user can make per second to prevent abuse. I used Spring Cloud Gateway's RequestRateLimiter filter with Redis backend. It uses the Token Bucket algorithm — each user gets 10 tokens per second in Redis. Each request consumes one token. When tokens run out, Gateway returns 429 Too Many Requests.

Q: What is the difference between GlobalFilter and GatewayFilter?

GlobalFilter applies to ALL routes automatically — used for JWT auth and logging that every request needs. GatewayFilter applies to specific routes only — used for rate limiter which has different limits per route. In our project, JWT uses GlobalFilter, rate limiter uses GatewayFilter defined per route in application.yml.

Q: Why does Gateway use WebFlux instead of regular Spring MVC?

Gateway handles thousands of simultaneous requests. Traditional Spring MVC is blocking — one thread per request. WebFlux is non-blocking — one thread handles many requests asynchronously. This makes Gateway extremely performant without using too many server threads.
